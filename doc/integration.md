je souhaite mettre en place un module de gestion de l’intégration administrative des agents afin de dématérialiser l’ensemble du processus d’entrée d’une personne au sein de l’administration.

Le module doit permettre de gérer :

* les différents modes d’intégration ;
* les dossiers administratifs des agents ;
* les validations hiérarchiques ;
* les contrats et décisions administratives ;
* les affectations ;
* les nominations ;
* la création des comptes utilisateurs ;
* la remise des moyens de travail ;
* l’historique administratif de chaque agent.

Le processus doit être entièrement traçable et conforme aux pratiques de la fonction publique et des établissements publics.

⸻

2. Objectifs du module

Le module doit permettre de :

1. Centraliser les dossiers d’intégration.
2. Standardiser les procédures administratives.
3. Réduire les délais de traitement.
4. Assurer la traçabilité des décisions.
5. Automatiser la production des documents.
6. Sécuriser les données des agents.
7. Constituer le dossier administratif numérique de chaque agent.

⸻

3. Types d’intégration

Le système doit permettre de gérer plusieurs types d’entrée dans l’administration.

3.1 Recrutement externe

Entrée d’une personne dans l’administration suite à :

* un concours ;
* un appel à candidature ;
* un recrutement direct.

Documents :

* décision de recrutement ;
* procès-verbal de recrutement ;
* note de service.

⸻

3.2 Mutation

Entrée d’un agent provenant d’une autre administration.

Documents :

* décision de mutation ;
* arrêté ministériel ;
* certificat de cessation de service.

⸻

3.3 Détachement

Affectation temporaire d’un agent provenant d’une autre administration.

Documents :

* arrêté de détachement ;
* accord de l’administration d’origine.

⸻

3.4 Mise à disposition

Mise à disposition d’un agent par une autre institution.

Documents :

* convention de mise à disposition ;
* décision administrative.

⸻

3.5 Réintégration

Retour d’un agent après :

* disponibilité ;
* détachement ;
* congé spécial ;
* suspension.

Documents :

* décision de réintégration.

⸻

3.6 Recrutement contractuel

Recrutement sous :

* CDD ;
* CDI.

Documents :

* contrat de travail.

⸻

3.7 Stage professionnel

Entrée d’un :

* stagiaire ;
* agent en période d’essai ;
* étudiant.

Documents :

* convention de stage.

5. Workflow détaillé

Étape 1 : Création de la demande

Informations :

* type d’intégration ;
* date de demande ;
* service demandeur ;
* poste demandé ;
* nombre de postes.

Pièces jointes :

* lettre de demande ;
* décision administrative ;
* note de service.

Statut :
BROUILLON
SOUMIS

Étape 2 : Validation RH

Contrôles :

* disponibilité du poste ;
* disponibilité budgétaire ;
* conformité du profil ;
* conformité des documents.

Décisions :

REJETÉ RH

10. Nomination

La nomination attribue une responsabilité hiérarchique.

Exemples :

* Directeur Général ;
* Directeur Central ;
* Directeur Départemental ;
* Chef de Service ;
* Chef de Bureau.

Règles :

* une structure ne peut avoir qu’un seul responsable actif ;
* une nomination possède une date de début et éventuellement une date de fin ;
* toute nouvelle nomination clôture automatiquement l’ancienne.

Documents :

* arrêté ;
* décision ;
* note de service.

11. Création du compte utilisateur

Après l’affectation :

Création automatique :

* compte application RH ;
* adresse e-mail professionnelle ;
* identifiant de connexion ;
* mot de passe temporaire ;
* badge.

Exemple :
Matricule : ARTF-2026-000125
Login : tlouppe
Email : tlouppe@artf.cg

12. Remise des moyens de travail

Matériels :

* ordinateur ;
* téléphone ;
* badge ;
* véhicule ;
* clés de bureau ;
* équipements de protection.

Document :

* fiche de remise de matériel ;
* procès-verbal.

13. Prise de service

Le responsable hiérarchique confirme :

* présence de l’agent ;
* installation du poste ;
* remise des équipements.

Document :

* procès-verbal de prise de service.
